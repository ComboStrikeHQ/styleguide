Tables

Standard Table

All table elements use the .table class.

Responsive Tables

Create responsive tables by wrapping any .table in .table-responsive.

Borders

Tables that take up the whole page/screen generally should be without borders. Tables that appear within the flow of other content or isolated on their own should have borders. Borders are added with the .table-bordered class on the .table element.

Stripes + Hover

Most tables should use the .table-striped and .table-hover classes (on the .table element), to aid legibility. Tables containing hidden rows should not be striped.

Tabs

Content Tabs

Content tabs are used to separate content into visually exclusive sections that can be accessed via a single content area.
They provide a flat navigation structure for content, without requiring a page refresh on change.

Content tabs should be used in cases where only one tab's content would be of individual interest to the user.
In cases where the user is expected to sequentially move through all tabs (rather than just selecting one), a wizard is more appropriate.

Toolbars

Toolbar Types

Three toolbar variations are available: Blank, Default and Primary.

The Primary toolbar is most common, and is used in order to draw attention to important inputs/filter functions (etc.).

The Default toolbar is for use-cases where the toolbar offers less-important or fully-optional functions. A secondary toolbar, for instance, should always be in the 'Default' style.

The Blank toolbar can be used in rare cases where toolbar behaviour (horizontal grouping of buttons/inputs) is desired, but not the visual separation/attention that comes with the toolbar background. This also can achieve the effect of 'floating' toolbar inputs.

The Drilldown toolbar is for filters, metrics, or other toggleable buttons. It uses the checkbox inputs to function, as seen in the code example.
